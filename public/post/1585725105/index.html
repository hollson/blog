1、什么是Nginx?

Nginx是一款开源的、高性能的HTTP服务器、反向代理服务器和邮件代理服务器,具有负载均衡器和HTTP缓存的能力。
Nginx使用Epoll作为IO多路复用机制，它是一种异步、非阻塞的事件驱动机制，很好的解决了C10K问题。

2、列举Nginx的特性

  • 跨平台：可以在大多数Unix like 系统编译运行。而且也有Windows的移植版本。
  • 配置简单：非常的简单，易上手。
  • 高并发：数据复制时，磁盘I/O的第一阶段是非阻塞的。官方测试能支持5万并发连接。
  • 无需长连接：Nginx代理和后端Web服务器间无需长连接；
  • 异步请求:Nginx接收用户请求是异步的，即先将用户请求全部接收下来，再一次性发送到后端Web服务器，极大减轻后端Web服务器的压力。
  • 响应快：发送响应报文时，是边接收来自后端Web服务器的数据，边发送给客户端。
  • 网络依赖性低: 理论上只要能够ping通就可以实施负载均衡，而且可以有效区分内网、外网流量。
  • 故障检测: Nginx能够根据应用服务器处理页面返回的状态码、超时信息等检测服务器是否出现故障，并及时返回错误的请求重新提交到其它节点上。
  • 成本低：内存消耗小、成本低廉（比F5硬件负载均衡器廉价太多）、节省带宽、稳定性高等特点。

4、Nginx如何处理HTTP请求?

多进程机制:
当收到一个客户端时，就由master进程生成一个worker进程来和客户端建立连接，直到连接断开，该子进程就结束了。
使用进程的好处是各个进程相互独立，无需加锁，同时降低编程的复杂度;缺点是生成子进程需要内存复制等操作，当有大量请求时，会导致系统性能下降 。

异步非阻塞机制:
每个工作进程使用异步非阻塞方式，可以处理多个客户端请求 。
当某个工作进程接收到客户端的请求以后，调用IO进行处理，如果不能立即得到结果，就去处理其他请求（即为非阻塞）;而客户端在此期间也无需等待响应，可以去处理其他事情（即为异步）。
当IO返回时，就会通知此工作进程 ；该进程得到通知，暂时挂起当前处理的事务去响应客户端请求 。

6、 反向代理的优点是什么?

1.隐藏真实服务器，提高了Web服务安全性。
2.可实现动静资源分离，加速了对网站访问速度。
3.可节约了有限的IP地址资源，服务器分配私有地址，采用虚拟主机的方式对外提供服务。

7、请列举Nginx最佳用途。

Nginx服务器的最佳用法是在网络上部署动态HTTP内容，使用SCGI、WSGI应用程序服务器、用于脚本的FastCGI处理程序。

8、Master和Worker进程分别是什么?

主程序Master process启动后，通过一个for循环来接收和处理外部信号 ；
主进程通过fork()函数产生worker子进程 ，每个子进程执行一个for循环来实现Nginx服务器对事件的接收和处理 。
一般推荐 worker 进程数与CPU内核数一致，避免进程之间竞争CPU资源和进程切换的开销。

对于每个请求，有且只有一个worker进程对其处理,每个worker进程都是从master进程fork过来。
当一个worker进程在accept这个连接之后，就开始读取请求、解析请求、处理请求，产生数据后，再返回给客户端，最后才断开连接。
在Nginx服务器的运行过程中，主进程和工作进程需要进程交互。交互依赖于Socket管道来实现。

9、什么是正向代理和反向代理?

正向代理是作为客户端的角色出现，在浏览器端配置，如VPN代理等。
反向代理部署在服务端，隐藏了实际服务器信息，是为服务器“代言”。

10、能否将Nginx的错误替换为502错误、503?

可以(502 =错误网关;503 =服务器超载), 但确保fastcgi_intercept_errors被设置为ON，并使用错误页面指令。

1
2
3
4
5

location /{
    fastcgi_pass 127.0.0.1:8080;
    fastcgi_intercept_errors on;
    error_page 502=503/error_page.html;
}

11、Nginx如何保留URL中的双斜线?

要在URL中保留双斜线，就必须使用merge_slashes_off;
语法:merge_slashes [on/off]

12、ngx_http_upstream_module的作用是什么?

ngx_http_upstream_module用于定义可通过fastcgi传递、proxy传递、uwsgi传递、memcached传递和scgi传递指令来引用的服务器组。

13、什么是C10K问题?

  • 进程是操作系统最昂贵的资源，单机不能无限创建TCP进程。如果超过1万个进程，那么操作系统是无法承受的。
  • 解决C10K瓶颈的思路就是IO多路复用，通过异步编程和事件触发机制替换轮训，IO 采用非阻塞的方式，减少不必要的性能损耗。底层的相关技术包括epoll、kqueue、libevent等，应用层面的解决方案包括OpenResty、Golang、Node.js等
  • 但C10K还需要考虑「最弱链接」问题。也就是说单机超过1万个链接时，哪怕只有一个要素没有考虑周到，都会导致问题的发生。

如果往两端用力拉一条由很多环 （连接）组成的锁链，其中最脆弱的一个连接会先断掉。因此，锁链整体的强度取决于其中最脆弱的一环。

14、stub_status和sub_filter作用是什么?

Stub_status：用于了解Nginx当前状态，如当前的活动连接，接受和处理当前读/写/等待连接的总数；
Sub_filter：它用于搜索和替换响应中的内容，并快速修复陈旧的数据；

15、Nginx是否支持将请求压缩到上游?

可以使用gunzip将请求压缩到上游。gunzip模块是一个过滤器，它可以对不支持“gzip”编码方法的客户机或服务器使用“内容编码:gzip”来解压缩响应。

16、如何获得当前的时间?

使用SSI模块、$date_gmt和$date_local的变量。Proxy_set_header THE-TIME $date_gmt;
该ngx_http_ssi_module模块是一个过滤器，用于处理通过它的响应中的SSI（服务器端包含）命令。

17、解释-s参数的作用是什么?

向master进程发送信号(send signal to a master process: stop, quit, reopen, reload)

18、如何在Nginx上添加模块?

安装编译时，加入要编译的模块，如--with-http_stub_status_module表示统计模块，类似的还有防盗链模块，哈希一致性、redis等模块。
