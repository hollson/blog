原子操作

原子操作中，针对某个值的操作，CPU不会再去进行其他的操作。为了实现这样的严谨性，原子操作仅会由一个独立的CPU指令代表和完成。原子操作是无锁的，常常直接通过 CPU 指令直接实现。 事实上，其它同步技术的实现常常依赖于原子操作。

Go 对原子操作的支持

sync/atomic 包提供了底层的原子级内存操作，用于同步访问整数和指针。

  • Go 语言提供的原子操作都是非入侵式的。
  • 原子操作共有五种：增减、比较并交换、载入、存储、交换。
  • 原子操作支持:int32、int64、uint32、uint64、uintptr、unsafe.Pointer。

func main() {
    var (
        n  int32
        wg sync.WaitGroup
    )
    // 每个goroutine都将整数n加1
    for i := 0; i < 1000; i++ {
        wg.Add(1)
        go func() {
            atomic.AddInt32(&n, 1)
            wg.Done()
        }()
    }
    wg.Wait()
    fmt.Println(atomic.LoadInt32(&n))
}

// num的值有可能不等于100，频繁修改变量值情况下，CAS操作有可能不成功。
func main() {
    var num int32
    wg := sync.WaitGroup{}
    for i := 0; i < 1000; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            tmp := atomic.LoadInt32(&num)
            if !atomic.CompareAndSwapInt32(&num, tmp, tmp+1) {
                fmt.Println("sorry")
            }
        }()
    }
    wg.Wait()
    fmt.Println("num : ", num)
}

func main() {
    // 交换操作,直接设置新值，返回旧值
    var num int32
    wg := sync.WaitGroup{}
    for i := 0; i < 10; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            tmp := atomic.LoadInt32(&num)
            old := atomic.SwapInt32(&num, tmp+1)
            // old := atomic.SwapInt32(&num, num+1)  //单原子操作
            fmt.Println("旧值 : ", old)
        }()
    }
    wg.Wait()
    fmt.Println("num : ", num)
}

原子操作与互斥锁的区别

  • 互斥锁是一种数据结构，使你可以执行一系列互斥操作。而原子操作是互斥的单个操作;
  • atomic的操作更轻量，比如CAS可以在不形成临界区和创建互斥量的情况下完成相关操作，大大的减少同步对程序性能的损耗。
  • 原子操作也有劣势,如CAS操作类似乐观锁，总是假设被操作值未曾被改变，那么在被操作值被频繁变更的情况下，操作成功率较低。而使用互斥锁(悲观锁)，将相关操作放入临界区中加以保护，将会更加高效。

原子操作使用规则

  • 不能用 atomic.Value 原子值存储 nil
  • 我们向原子值存储的第一个值，决定了它今后能且只能存储哪一个类型的值
  • 不要把内部使用的atomic.Value原子值暴露给外界，如果非要暴露也要通过 API 封装形式，做严格的 check。
    对于应用层来说，最好使用通道或 sync 包中提供的功能来完成同步操作。
